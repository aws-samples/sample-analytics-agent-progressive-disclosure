# 分析方法 · 贡献度分析 + 帕累托 (Contribution & Pareto)

> 套路：一个总量由很多子项（渠道/品类/地区/套餐）构成，找出**谁贡献最大、谁在变化、
> 是否符合二八定律**。比"列出每项数值"高级——它回答"重点抓哪几个"。

## 何时用
"哪些渠道贡献了大部分 GMV""增长主要来自哪些品类""二八分布""集中度"等。

## 统计公式（帕累托 + 集中度，请照此算并填进 method）
设各子项值降序为 v₁≥v₂≥…≥vₖ，总和 V = Σ vᵢ：
- **占比**：shareᵢ = vᵢ / V
- **累计占比**：cum_j = (Σ_{i≤j} vᵢ) / V —— 帕累托曲线纵轴。
- **80% 拐点**：最小的 j 使 cum_j ≥ 80%，这 j 个即核心贡献者。
- **集中度 CR_n**：前 n 项占比之和（如 CR₃ = 前3项占比），衡量依赖集中程度。
- **HHI（赫芬达尔指数）**：HHI = Σ shareᵢ² ∈ (0,1]，越接近 1 越集中/依赖单一子项。

## 做法（资深 SOP；这几步填进 present_result.method.steps）
1. **取构成**：按维度（如渠道）拿各子项的指标值（call_metric gmv_by_channel group_by channel，或简单 SELECT）。
2. **排序 + 累计占比**：降序排列，算每项 share 和**累计占比 cum**（帕累托核心）。
3. **找 80% 线**：累计到 80% 是哪几项 = 核心贡献者；其余是长尾（把 CR₃/HHI/80%需要几项填进 method.stats）。
4. **变化贡献**（若问"增长来自谁"）：算各子项**本期 vs 上期的增量**，按增量排序，找出拉动增长/拖累的子项。
5. **必看未归因**：渠道分析时"未归因"约占六成 GMV，必须单列、披露占比，不能假装不存在。

## 输出要求（多图）
- **charts 给 2 个**：
  1. hbar：各子项指标值降序（categories + values + note 放占比）
  2. line 或 bar：累计占比帕累托曲线（可在 note 标 80% 线），或各子项增量对比
- KPI：核心贡献者数量（如"前3渠道占72%"）、第一名占比、未归因占比。
- findings：driver（核心贡献子项及占比）、risk（集中度过高的依赖风险 / 未归因占比过大）。
- insight：点出"X、Y、Z 三项占了 N%，是重点；其余是长尾"。

## 口径提醒
- 占比分母要说清（含不含未归因，差别很大）。
- 帕累托的"80%"是经验线，不是硬规则，按实际拐点说。
