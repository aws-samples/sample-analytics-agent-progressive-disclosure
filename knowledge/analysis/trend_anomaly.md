# 分析方法 · 趋势 + 同环比 + 异常检测 (Trend & Anomaly)

> 套路：看一个指标不要只给"当前值"，要放进时间序列里看**趋势、对比基准、异常点**。

## 何时用
"最近怎么样""有没有异常""这周 vs 上周""DAU 趋势"等任何带时间的诊断题。

## 时间锚点（务必遵守）
本库静态样本，最新数据约 2026-01-24。一律以**表自身 dt 的 max() 作为"今天"**，
严禁 current_date/now()（会查空）。相对周期都相对该锚点算。

## 统计公式（异常检测用 Z-score / 3σ 准则，请照此算）
设日序列为 x₁…xₙ：
- **均值**：μ = (Σ xᵢ) / n
- **标准差**：σ = √( Σ(xᵢ − μ)² / n )
- **Z 分数（标准分）**：Zᵢ = (xᵢ − μ) / σ —— 表示某天偏离均值几个标准差。
- **判定阈值**：|Zᵢ| > 2 视为异常（约覆盖正态分布的 95%）；|Zᵢ| > 3 为强异常（3σ 准则，99.7%）。
- **变异系数 CV**：CV = σ / μ —— 衡量整体波动幅度（CV 越大越不稳）。
- **环比**：MoM = (本期 − 上期) / 上期。
> 也可用移动均线/中位数±MAD 抗离群，但 demo 用 Z-score 最直观、可解释。

## 做法（异常检测 SOP，把这几步填进 present_result.method.steps）
1. **取序列**：用 `call_metric(dau, group_by=["date"], time_window="last_30d")` 之类拿日序列。
2. **算 μ 和 σ**：对序列求均值 μ、标准差 σ（把算出的 μ/σ/CV 填进 method.stats）。
3. **逐点算 Z 分数**：Zᵢ = (xᵢ − μ) / σ。
4. **按阈值判异常**：|Z| > 2 标为异常点；记录是哪天、偏离几个 σ、偏离百分比。
5. **定位 + 归因**：对异常日结合业务（大促/节假日/投放/数据缺口）给可能原因。
   - chart 的 series 里给 `markPoints`（异常点坐标 {coord:[x,y],name}）和 `markLines`（μ 均值线、μ±2σ 阈值带），前端高亮。
   - 异常写成 `findings`：`{kind:'anomaly', text:'01-18 DAU=xxx，Z=-2.4（低于均值2.4个σ），环比-28%', evidence:'μ=..., σ=...'}`。
6. **下判断**：趋势升/降、拐点在哪、异常是否值得追，而不是罗列每天的数。

## 输出要求
- chart 用 line（带 markPoint/markLine）。
- KPI：当前值 + delta（环比）+ baseline 对比基准。
- findings：anomaly 项写清异常点和偏离幅度；若趋势平稳无异常，如实说"无显著异常"。
- 残周/残月、节假日效应等不确定性放进 risk finding。
