# 分析方法 · 比率拆解 / 驱动归因 (Ratio Decomposition)

> 资深分析师的可复用套路：一个**总量指标动了**，不要只报"涨了/跌了 X%"，要把它拆成
> 乘法因子，定位**到底是哪个因子在驱动**。这是 text-to-insight 比"一个数"高级的核心。

## 何时用
问题形如"GMV 为什么涨/跌""转化为什么下降""收入波动是谁带动的"——任何总量指标的归因。

## 核心拆解公式（电商）
- **GMV = 付费用户数 × 客单价**（客单价 = GMV / 付费用户数）
  → GMV 跌了，先看是付费人数跌（拉新/流量问题）还是客单价跌（折扣/品类结构问题）。
- **GMV = 订单数 × 平均订单金额**；**订单数 = 付费用户数 × 人均下单次数**。
- **新客 GMV = 新客数 × 新客客单价**；用 mart_daily_revenue 的 is_new_user 切。
- **渠道 GMV = Σ(各渠道 GMV)**：按 channel 切，找出贡献变化最大的渠道（含"未归因"）。

## 做法（多角度连发，不要一条 SQL 完事）
1. 先用 `call_metric` 或简单 SELECT 拿总量指标的两个周期值（本期 vs 上一周期，整周/整月）。
2. 把总量按公式拆成因子，分别取两期值，算**各因子的变化率**。
3. 用「变化贡献」定位主因：哪个因子的变化方向和幅度最能解释总量的变化。
4. 写成 `findings`：`{kind:'driver', text:'GMV 环比 -12%，主要由付费用户数 -15% 拉低，客单价基本持平(+2%)', evidence:'...'}`。

## 统计公式（两因子分解，请照此算并填进 method）
设 GMV = U × P（U=付费用户数, P=客单价），下标 0=上期、1=本期：
- **总变化**：ΔGMV = U₁P₁ − U₀P₀
- **因子贡献（对称/中点分解，避免顺序偏差）**：
  - 用户数贡献：ΔU × (P₀ + P₁) / 2
  - 客单价贡献：ΔP × (U₀ + U₁) / 2
  - 两者相加 = ΔGMV（分解无残差，便于展示）
- **贡献占比**：某因子贡献 / ΔGMV × 100%
- **各因子变化率**：ΔU/U₀、ΔP/P₀
> 简化近似（够用）：因子A贡献 ≈ ΔA × B₀，因子B贡献 ≈ A₁ × ΔB。中点法更精确、无残差，优先用。

## 资深分析师 SOP（按步走，别跳；这几步填进 present_result.method.steps）
1. **大盘**：先 call_metric 或简单 SELECT 拿总量指标本期 vs 上一周期两个值，算总变化率 ΔGMV/GMV₀。
2. **拆因子**：按乘法公式取每个因子两期值 U₀U₁P₀P₁，分别算变化率。
3. **量化贡献**：用上面的中点分解算各因子对 ΔGMV 的贡献额和贡献占比（把 ΔGMV、各因子贡献额/占比填进 method.stats）。
4. **下钻验证**：主驱动因子再按一个维度（渠道/新老客）切，定位是全局变化还是某一档带动。
5. **结论 + 行动**：哪个因子驱动、健康度如何、下一步该做什么。

## 输出要求（用多图，体现深度）
- **charts 给 2-3 个图**（这是深度分析，别只给一张）：
  1. line：总量指标两期趋势对比（或本期日序列）
  2. bar：两期各因子对比（付费用户数、客单价并排），直观看谁变了
  3. bar/pie：各因子对总变化的**贡献占比**（最能体现"归因"）
- KPI 卡片给总量 + 关键因子，每个带 delta + baseline（上期值）。
- findings 必出：driver（主驱动因子+贡献占比%）、anomaly（若有突变）、risk（健康度隐忧，如"全靠堆量、单价不涨"）。
- insight 一句话点出"是X驱动而非Y"。
- **诚实**：残周/残月不直接比；样本太小要在 risk 说明。
